#  电商用户行为分析与用户分层

## 项目概述

基于某电商平台 300 万条用户行为日志数据，围绕**用户购物行为路径**进行系统性分析，识别用户在点击 → 收藏 → 加购 → 购买各环节的转化特征与流失环节，并运用改进型 RFM 模型对用户进行价值分层，最终结合时间维度分析用户活跃规律，为电商运营策略优化提供数据驱动的决策支撑。

## 技术栈

- **Python 3**
- **NumPy** — 数值计算与数据处理
- **Pandas** — 数据清洗、转换、聚合与统计分析
- **Tableau** — 数据可视化

## 项目内容

### 1. 数据清洗
- 手动添加表头（原始 CSV 无表头）
- 缺失值检查：**无缺失值**
- 重复值处理：删除 19 条重复记录
- 时间戳转换：Unix 时间戳 → datetime 格式
- 行为文本替换：`pv→点击`、`buy→购买`、`cart→加入购物车`、`fav→收藏商品`
- **清洗结果**：有效数据 **2,999,981 条**，覆盖 **29,178 名用户**、**805,067 件商品**、**6,868 个商品类目**，时间跨度为 2017-07-03 至 2017-12-06

### 2. 用户行为总览
- **行为分布**：点击占 89.5%，加购占 5.6%，收藏占 2.9%，购买占 2.0%
- **人均行为数**：人均点击 92 次、加购 6 次、收藏 3 次、购买 2 次
- **用户活跃度**：中位数 77 次/人，中间 80% 的用户活跃次数集中在 **22 ~ 221 次**之间

### 3. 转化漏斗分析
- **路径优化**：原始路径为「点击→收藏→加购→购买」，但发现收藏人数（11,701 人）低于加购人数（22,061 人），不符合漏斗递减规律，说明收藏并非必经之路，**将路径修正为「点击→加购→购买」**
- **转化率**：

| 环节 | 用户数 | 环比转化率 | 整体转化率 |
|:---:|:-----:|:--------:|:--------:|
| 点击 | 29,049 | - | 100% |
| 加入购物车 | 22,061 | 75.9% | 75.9% |
| 购买 | 19,902 | 90.2% | 68.5% |

- **流失分析**：
  - 点击→加购：流失 6,988 人，流失率 **24.1%** ← **最大流失环节**
  - 加购→购买：流失 2,159 人，流失率 **9.8%**
- **业务建议**：点击→加购环节是最大瓶颈，说明商品详情页或宣传信息未能充分吸引用户，存在优化空间

### 4. 用户分层分析（改进型 RFM 模型）

由于数据中无购买金额字段，将传统 RFM 模型中的 **M（消费金额）** 替换为**行为加权得分**（点击=1分，收藏/加购=2分，购买=3分），构建改进型 RFM 模型：

| 维度 | 定义 | 权重 |
|:---:|:---:|:---:|
| R | 距离最近一次购买的天数（未购买赋 999） | 0.3 |
| F | 购买次数（未购买赋 0） | 0.3 |
| M | 行为加权总得分（衡量用户活跃程度） | 0.4 |

**分层结果**（基于综合得分四等分）：

| 等级 | 用户数 | 含义 |
|:---:|:-----:|:----|
| A | 6,728 | 高价值活跃用户 |
| B | 7,701 | 潜力价值用户 |
| C | 6,704 | 一般用户 |
| D | 8,045 | 低活跃/流失风险用户 |

> 针对 R、F 分数分布两极分化的问题，将分箱策略从五箱调整为三箱，使分层更加合理均衡。

### 5. 时间维度分析
- **11 月日均活跃用户**：5,540 人
- **周活跃度**：除周五外（数据采样偏差导致），各日活跃用户数较为均衡（20,889 ~ 21,810 人）
- **时段活跃度**：
  - 早高峰：5:00~9:00（66,610 人）
  - 午高峰：11:00~15:00（71,234 人）
  - 晚高峰：22:00~4:00（59,538 人）
- **业务建议**：建议在上述高峰时段进行消息推送和运营活动

## 核心结论

1. **点击→加购环节是最大流失点**（流失率 24.1%），建议优化商品展示页
2. **收藏并非用户必经行为**，不应作为漏斗必经环节
3. **高价值用户（A 级）占 23%**，可针对其进行精细化运营，提升复购率
4. **D 级用户（低活跃/流失风险）占比 27.6%**，需关注召回策略
5. 用户活跃存在明显的**早、午、晚三个高峰时段**，运营推送应集中于此

## 数据说明

- 原始数据来源于阿里云天池平台（UserBehavior.csv），总量约 1 亿条
- 本项目随机抽取前 300 万条进行分析
- 数据字段：用户 ID、商品 ID、商品类目 ID、用户行为类型、发生时间戳
- 下载数据集：前往 https://tianchi.aliyun.com/dataset/ 搜索 UserBehavior 数据集
